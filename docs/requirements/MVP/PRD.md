# 简约博客项目 PRD（产品需求文档）

## 1. 项目概述
- **项目名称**：简约博客（暂定名）
- **部署目标**：GitHub Pages (`k1he.github.io`)
- **核心原则**：
  - 简约轻量：无构建步骤，纯前端实现
  - 视觉优先：美观的科技/代码风格
  - 功能聚焦：MVP 版本仅包含文章展示

## 2. 功能需求（MVP）

### 2.1 核心功能
- **文章列表页**：展示所有文章，按发布时间倒序排列
  - 每篇文章显示：标题、发布日期、摘要（前200字符）
  - 点击标题进入文章详情页
- **文章详情页**：完整展示单篇文章内容
  - 从 Markdown 文件渲染为 HTML
  - 支持代码高亮、数学公式、图表、图片灯箱
- **侧边栏导航**：固定侧边栏，包含：
  - 博客标题/Logo（点击返回首页）
  - 导航菜单（首页、归档、关于 - 页面待实现）
  - 文章分类/标签（动态从文章元数据提取）

### 2.2 内容管理
- 文章以 Markdown 文件存储，放置在 `posts/` 目录
- 每篇文章的元数据（标题、日期、标签等）使用 Front Matter
- 无后端，所有文件静态托管

### 2.3 技术增强
- ✅ 代码高亮（支持多种语言）
- ✅ 数学公式渲染（LaTeX）
- ✅ 图表支持（Mermaid）
- ✅ 图片灯箱（点击放大）
- ✅ 响应式设计（移动端友好）

## 3. 设计需求

### 3.1 视觉风格
- **主题**：科技/代码风
- **字体**：
  - 正文：等宽字体（如 `'SF Mono', 'Monaco', 'Consolas', monospace`）
  - 标题：稍粗的等宽字体或几何无衬线字体
- **色彩方案**：
  - 主色调：深色背景（`#0d1117` 类似 GitHub Dark）
  - 文字色：浅灰（`#c9d1d9`）
  - 强调色：青色/蓝色（`#58a6ff` 类似 GitHub 链接色）
  - 代码背景：深灰（`#161b22`）
  - 边框：`#30363d`

### 3.2 布局
- **侧边栏导航**：左侧固定宽度（约 240px），跟随滚动
- **主内容区**：右侧自适应宽度，最大宽度约 900px
- **文章列表**：标题 + 摘要形式，每项之间有分隔线
- **文章详情**：居中对齐阅读区域，合适的行长（约 80 字符宽度）

### 3.3 交互细节
- 链接悬停效果
- 平滑滚动
- 侧边栏当前页面高亮
- 移动端侧边栏可收起/展开

## 4. 技术方案

### 4.1 核心架构
采用 **纯前端 + 客户端渲染** 方案：
1. 所有 Markdown 文章作为静态文件放置在 `posts/` 目录
2. 使用 JavaScript 在浏览器中：
   - 读取文章列表（通过 `fetch` 获取目录索引或扫描 `.md` 文件）
   - 解析 Markdown 和 Front Matter
   - 动态生成 HTML 内容
3. 无需构建工具，直接上传到 GitHub Pages

### 4.2 技术栈
- **Markdown 解析**：[marked.js](https://marked.js.org/)（轻量、易用）
- **Front Matter 解析**：[front-matter](https://github.com/jxson/front-matter) 或自定义解析
- **代码高亮**：[Prism.js](https://prismjs.com/)（主题丰富，支持行号）
- **数学公式**：[KaTeX](https://katex.org/)（渲染速度快）
- **图表**：[Mermaid.js](https://mermaid.js.org/)（流程图、时序图等）
- **图片灯箱**：[medium-zoom](https://github.com/francoischalifour/medium-zoom) 或简单 Lightbox 实现
- **CSS 框架**：自定义 CSS，辅以少量工具类

### 4.3 数据流
```
用户访问网站 → 加载 index.html + CSS + JS
→ JS 读取 posts/ 目录下的 .md 文件列表
→ 解析每篇文章的 Front Matter 获取元数据
→ 生成文章列表页面
→ 点击文章 → 加载对应 .md 文件 → 解析渲染 → 显示详情页
```
